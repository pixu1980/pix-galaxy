# Support

## When to Open an Issue

- Use the bug report template for broken behavior, regressions, packaging problems, or release issues.
- Use the feature request template for API ideas, package proposals, or documentation improvements.

## Before Opening an Issue

- Read README.md.
- Check existing issues and pull requests.
- Reproduce on the latest published version when possible.

## Security Issues

For vulnerabilities, follow SECURITY.md and avoid public disclosure before maintainers can triage the report.

## Scope of Support

The project supports the published packages, documented API surface, and maintained examples in the repository. Help with unrelated application code, framework-specific integrations, or custom forks may be limited.
